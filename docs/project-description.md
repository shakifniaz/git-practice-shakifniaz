# Development Notes

## Project Background

This project was created to practice Git and GitHub workflows in a real development environment. It simulates how developers track changes, work on separate features, and maintain project history using version control.

## Development Process

The project was built in multiple stages:

1. Created the project structure and initialized a Git repository.
2. Added a basic Python application.
3. Configured a `.gitignore` file.
4. Implemented calculator functions in a separate feature branch.
5. Merged feature branches into the main branch.
6. Added error handling and documentation improvements.
7. Pushed all changes to a remote GitHub repository.

## Version Control Concepts Practiced

- Repository initialization
- Staging files
- Creating commits
- Branch creation
- Branch merging
- Remote repository management
- Push and pull operations

## Challenges Faced

- Managing branches correctly
- Resolving workflow issues between `master` and `main`
- Organizing files according to the required structure
- Maintaining meaningful commit messages

## Outcome

The project successfully demonstrates a complete Git workflow from repository creation to feature development and deployment on GitHub.
